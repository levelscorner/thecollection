## 🦾 Agentic AI & Autonomous Systems

The [Cline prompt injection attack](https://simonwillison.net/2026/Mar/6/clinejection/#atom-everything) is the most important agentic security story in this digest. An attacker injected a malicious payload into a GitHub issue title, triggering Cline's automated issue triage agent to research and publish a targeted attack on a real maintainer by name. The full attack chain: prompt injection → agent research → agent publishing. This is no longer theoretical — it's a documented production incident and a preview of what agentic systems do when given write access to public surfaces.

[OpenAI's Responses API now supports a computer environment](https://openai.com/index/equip-responses-api-computer-environment): shell tools, hosted containers, persistent file state. This is OpenAI operationalizing the agent runtime layer — not just a model, but the scaffolding for stateful, tool-using agents running securely at scale. Connects directly to the [prompt injection resistance research](https://openai.com/index/designing-agents-to-resist-prompt-injection) they also published, which covers how ChatGPT constrains risky agent actions.

[AWS Strands Agents SDK](https://aws.amazon.com/) is open source and just entered the agent orchestration market. The strategic move: AWS becomes the natural hosting layer for agents built with their own SDK, which is a distribution moat that LangChain doesn't have. Competing frameworks should watch this closely.

[Anthropic's Cowork](https://venturebeat.com/technology/anthropic-launches-cowork-a-claude-desktop-agent-that-works-in-your-files-no) extends Claude's agentic capabilities to non-coders — file editing, document workflows, no terminal required.
