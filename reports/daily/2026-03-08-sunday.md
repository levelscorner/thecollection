## 🧠 AI Research & Papers

[**CoT-Control research (OpenAI)**](https://openai.com/index/reasoning-models-chain-of-thought-controllability): Reasoning models resist external attempts to manipulate their chain-of-thought. The key insight: this resistance is a safety property, not a bug. If CoT were easily steered, it would be useless as a monitorability signal. Practitioners should note this has direct implications for jailbreaks and prompt injection attacks on reasoning models — they're harder to redirect mid-think.

[**LLM Evaluation: 4 Approaches (Raschka)**](https://magazine.sebastianraschka.com/p/llm-evaluation-4-approaches): Covers Multiple-Choice Benchmarks, Verifiers, Leaderboards, and LLM Judges with code. As AI math benchmarks are [saturating faster than researchers can write them](https://spectrum.ieee.org/ai-math-benchmarks), this taxonomy becomes critical for teams building evaluation pipelines.

[**Inference-Time Scaling (Raschka)**](https://magazine.sebastianraschka.com/p/categories-of-inference-time-scaling): Survey of scaling categories at inference time. The key practical tension: inference-time compute is where the next capability gains live, but it directly drives up per-query costs — which connects directly to the Claude Code pricing debate in Industry News.

[**KV Cache from Scratch (Raschka)**](https://magazine.sebastianraschka.com/p/coding-the-kv-cache-in-llms): Genuinely useful implementation guide for engineers building or optimizing inference stacks. KV cache management is now a core systems skill for anyone running LLMs in production.

[**LLM pseudonym unmasking (Ars Technica)**](https://arstechnica.com/security/2026/03/llms-can-unmask-pseudonymous-users-at-scale-with-surprising-accuracy/): LLMs can now de-anonymize pseudonymous users at scale. This is a privacy inflection point — the writing-style analysis that was once a forensic curiosity is now automatable. Anyone operating pseudonymous communities or user bases under GDPR should be paying attention.
